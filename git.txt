git branch name - Создание ветки name
git checkout name - Переключение на ветку name

git chechout -b name - Создание и переключение на ветку name

git branch - Вывод список локальных веток
git branch -a - Вывод всех веток

git log --graph - рисует ascii-граф взаимосвязи коммитов
git log --a - Выводит историю всех имеющихся коммитов

git diff <коммит> <коммит> - Показывает изменения коммитов

{
	git checkout master - Переход на master
	git merge name - Слияние
	git branch -d name - Удаление ветки name

} - Слияние ветки name с master

При конфликте после слияния в конфликтующие файлы будут добавлены строки по типу:

<<<<<<< HEAD:index.html
<div id="footer">contact : email.support@github.com</div>
=======
<div id="footer">
 please contact us at support@github.com
</div>
>>>>>>> name:index.html

Чтобы решить конфликт требуется оставить только один из предложенных вариантов:

<div id="footer">contact : email.support@github.com</div>

или

<div id="footer">
 please contact us at support@github.com
</div>

Либо объединить их как-то.

Все остальные строки следует удалить.

После разрешения конфликта запускаем git add, подтверждая тем самым решение конфликта.
git status - Проверки наличия других конфликтов.
git commit -m "text" - Создание коммита.
